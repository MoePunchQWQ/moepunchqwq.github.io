前言

  • 本教程安装 Appx/Appxbundle(UWP) 软件的方法仅仅是为安装此类型软件提供一个方便的环境（不需要手动安装依赖），恕不提供此类型软件下载。
  • 本教程安装 xap 软件的方法仅仅是绕开 Windows 10 Mobile 只能安装 20 个此类型软件的限制，恕不提供此类型软件下载。
  • 本教程不适用于 WP8/WP8.1

安装 Appx/Appxbundle(UWP) 软件的方法

系统要求

  • 一部能联网（不要求公网）并且系统版本号在 Windows 10 14393 ( Windows 10 Mobile 1607 ) 或以上的 Windows Phone 设备
  • 另一部能与 Windows Phone 设备连接同一网络并安装有浏览器的设备

Windows Phone 端操作

打开设置，翻到最底，点击更新与安全-针对开发人员，在使用开发人员功能条目里勾选开发人员模式，在设备发现条目里勾选允许 USB 连接和本地网络发现你的设备，在设备门户中打开启用通过 USB 和局域网连接进行的远程诊断，关闭其中的身份验证选项。记下连接时使用的 IP 地址。（ eg:https://192.168.114.514 ）

其他设备操作

确保与 Windows Phone 设备连接到同一局域网，打开浏览器，在地址栏中输入上文提到的 IP 地址，回车进入。你应该可以看见如下的画面。
点击 App package 中的选择文件，选中下载下来的文件中位于 DesktopAppInstaller 文件夹中的 DesktopAppInstaller.AppxBundle ，再在下面的 Dependency 中点击 Add Dependency ，选中 Dependencies 文件夹里的各个 .appx 文件（注：需要多次点击 Add Dependency 并选择文件）。如果你操作正确，你应该可以看见以下画面。
点击 Deploy 中的 Go 。等待片刻，网页刷新以后你的手机中就装上了应用安装程序，这个程序可以在你安装应用的时候自动安装所需的依赖，即上文提到的 Dependency 。接下来你可以将后缀名为 appx/appxbundle 的软件传输到手机中，使用文件资源管理器打开并安装（安装方式如图）。

安装 xap 软件的方法

系统要求

  • 一部系统版本号在 Windows 10 10586（ Windows 10 Mobile 1511 ） 或以上的 Windows Phone 设备
  • 一台最好是搭载 Windows 的电脑（本人使用的是 Windows 10 ，其他操作系统连接 Windows Phone 的教程恕不阐述，请自行寻找方法）

电脑端操作

将 Windows Phone 与电脑用数据线连接，确保电脑的文件资源管理器中有你的 Windows Phone 设备。
进入手机的数据目录，将下载文件中 Mobile Enterprise 文件夹中的两个文件复制到手机目录。

手机端操作

点击两次是的，请添加它，完毕后重启手机。
重启后，手机的系统版本会变为 Windows 10 移动企业版。此时可以连接电脑，用 Windows Phone SDK 给你的手机安装 xap 软件。
